상속(Inheritance)

	부모 클래스에 있는 멤버들을 자식 클래스에서 자유롭게 활용하고 싶을 때
		사용하는 문법
		>부모의 코드를 자식이 물려받아 사용하는 것

	메소드 오버라이딩(Overriding)
		상속 과정에서 메소드를 자식에서 재정의하는 것
		부모타입의 객체에서 메소드가 사용되면 부모에서 정의된대로 실행이 되고
		자식타입의 객체에서 메소드가 사용되면 자식에서 재정의된대로 실행이 된다.

	*오버로딩: 메소드의 이름은 같지만 매개변수의 타입/개수를 다르게 정의하는 것

	
	오버라이딩 조건
		1.같은 함수(이름, 매개변수, 리턴타입)이어야 한다.

		2.접근제어자를 좁은 범위로 변경 할 수 없다. public -> default로는 안됨
						       default -> public은 됨

		3.부모클래스의 메서보드보다 많은 수의 예외를 선언 할 수 없다.


	참조변수
		this -> 인스턴스 자신을 가르키는 참조변수

		super -> 부모클래스를 의미
			super.변수명, super.메소드면 형식으로 사용한다.

@~:어노테이션 주석
	컴퓨터가 보는 설명
	의도적으로 오류를 발생시킴으로서 실수를 줄일 수 있다.
	가독성을 높힐 수 있다.
		
오버라이딩은 다형성을 구현하는 요소이다.

	다형성(polymorphism)★★★
		:하나의 모양으로 코드가 다르게 동작하는 것

-상속과 생성자
	자식 클래스의 생성자 작업을 할 때 부모 클래스의 생성자도 반드시 호출 해주어야한다.

	1.super()
	부모 클래스의 생성자 호출은 상위 클래스를 의미하는 super에 ()를 붙인 super()를 통해 이루어진다.

	부모 클래스의 호출은 무조건 자식 클래스 생성자 첫 줄에서 이루어진다.

	만약, 자식 생성자 내부에 부모 클래스의 생성자를 따로 작성하지 않았다면
		자동적으로 컴퓨터가 자식 클래스의 생성자 첫줄에 super();를 추가한다.

	당연히 초기화 생성자도 사용과 호출이 가능하다.

	>>상속 후 객체화 과정에서 오류가 난다면 부모의 기본 생성자가 정의되어있는지 확인한다.


-Object 클래스
	자바의 최상위 클래스
		>Object 클래스는 모든 클래스의 부모 클래스이다.

	상속 계층도의 취상위에 Object 클래스가 위치한다.
		>누구의 상속도 받지 않는다.

		>모든 클래스는 Object 클래스에 정의된 11개의 메서드를 상속 받는다.
			ex)toString(),equlas(),.. 


-캐스팅(Casting): 형 변환

	업 캐스팅
		자식타입을 부모타입으로
		업캐스팅이 된다면 자식에서 새롭게 정의한 멤버는 사용이 불가능
		그러나 자식에서 재정의한 메소드는 재정의한대로 실행이된다.



	다운 캐스팅
		부모타입을 자식타입으로
		업캐스딩 된 객체를 다시 자식타입으로 되돌리는 행위!!!
		사용하지 못했던 멤버들을 다시 활용할 수 있다.
